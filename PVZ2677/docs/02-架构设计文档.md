# 植物大战僵尸 Lite — 架构设计文档（Architecture Design）

> 版本：v1.0-draft  
> 对应文档：[01-功能文档](./01-功能文档.md) | [需求文档](./EasyX%20暑期课程大作业需求文档.docx)  
> 技术栈：C++17 | EasyX | MSVC v143 | Windows

---

## 1. 设计目标

本架构设计遵循以下原则：

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个类仅负责一个明确的功能领域 |
| **Update/Render 分离** | 所有逻辑在 `Update(float dt)`，所有绘制在 `Render()`，禁止混杂 |
| **依赖倒置** | 高层模块不依赖低层模块，通过接口/管理器解耦 |
| **开闭原则** | 新增植物/僵尸类型只需新增子类，不修改已有代码 |
| **资源统一管理** | 所有图片/音频/字体由 `ResourceManager` 集中加载与释放 |
| **增量开发友好** | 模块间低耦合，可独立开发、测试、替换 |

## 4. 核心类设计

### 4.1 Game（游戏主控）

```
┌─────────────────────────────────────┐
│               Game                  │
├─────────────────────────────────────┤
│ - sceneManager: SceneManager*       │
│ - resourceManager: ResourceManager* │
│ - inputManager: InputManager*       │
│ - soundManager: SoundManager*       │
│ - gameTimer: GameTimer*             │
│ - config: Config*                   │
│ - isRunning: bool                   │
├─────────────────────────────────────┤
│ + Init(): void                      │
│ + Run(): void                       │
│ + Quit(): void                      │
│ + GetSceneManager(): SceneManager*  │
│ + GetResourceManager(): ResMgr*     │
│ + GetInputManager(): InputMgr*      │
│ + GetSoundManager(): SoundMgr*      │
│ + GetConfig(): Config*              │
└─────────────────────────────────────┘
```

**职责：**
- 程序入口 `main()` 创建 Game 单例并调用 `Run()`
- 初始化所有管理器
- 管理主循环（输入 → 更新 → 渲染）
- `Game` 采用单例模式，各模块通过 `Game::GetInstance()` 访问全局管理器

**主循环伪代码：**

```cpp
void Game::Run() {
    while (isRunning) {
        float dt = gameTimer->Tick();       // 计算 delta time
        inputManager->Update();             // 处理输入
        sceneManager->Update(dt);           // 更新当前场景
        BeginBatchDraw();                   // EasyX 双缓冲开始
        cleardevice();
        sceneManager->Render();             // 渲染当前场景
        EndBatchDraw();                     // EasyX 双缓冲结束
    }
    Quit();
}
```

### 4.2 SceneManager（场景管理器）

```
┌──────────────────────────────────────┐
│           SceneManager               │
├──────────────────────────────────────┤
│ - scenes: map<SceneID, Scene*>       │
│ - currentScene: Scene*               │
│ - pendingTransition: SceneID         │
├──────────────────────────────────────┤
│ + RegisterScene(id, scene): void     │
│ + SwitchTo(id): void                 │
│ + Update(dt): void                   │
│ + Render(): void                     │
│ + GetCurrentScene(): Scene*          │
└──────────────────────────────────────┘
```

**职责：**
- 持有所有场景实例的注册表
- 管理场景切换（延时切换，避免在 Update 中直接切换导致迭代器失效）
- 委托 `Update(dt)` / `Render()` 到当前场景

**场景切换流程：**

```
SwitchTo(newScene)
    → pendingTransition = newScene
    → 下一帧 Update 开头执行实际切换：
        → currentScene->OnExit()
        → currentScene = scenes[pendingTransition]
        → currentScene->OnEnter()
        → pendingTransition = NONE
```

### 4.3 Scene（场景基类）

```
┌──────────────────────────────────────┐
│          Scene (abstract)            │
├──────────────────────────────────────┤
│ # sceneID: SceneID                   │
├──────────────────────────────────────┤
│ + OnEnter(): void          (virtual) │
│ + OnExit(): void           (virtual) │
│ + Update(dt): void         (pure)    │
│ + Render(): void           (pure)    │

└──────────────────────────────────────┘

每个具体场景（MenuScene、GameplayScene 等）继承此类，实现 `Update` 和 `Render`。

```
┌──────────────────────────────────────┐

## 5. GameplayScene 核心战斗数据流

```
Game::Run() 每帧
  │
  ├─ 1. InputManager.Update()          ← 读取鼠标键盘
  │
  └─ 2. SceneManager.Update(dt)
        │
        └─ GameplayScene::Update(dt)
              │
              ├─ 2.1 WaveManager.Update(dt)      ← 波次计时，生成僵尸
              │
              ├─ 2.2 阳光系统更新
              │     ├─ 自然阳光计时器
              │     ├─ 向日葵产能计时器
              │     └─ 阳光超时清理
              │
              ├─ 2.3 实体更新（按顺序）
              │     ├─ for each Plant  → plant.Update(dt)
              │     ├─ for each Zombie → zombie.Update(dt)
              │     └─ for each Bullet → bullet.Update(dt)
              │
              ├─ 2.4 CollisionManager 碰撞检测
              │     ├─ CheckBulletZombie(bullets, zombies)
              │     ├─ CheckZombiePlant(zombies, map)
              │     └─ CheckZombieHouse(zombies, boundary)
              │
              ├─ 2.5 死亡清理
              │     ├─ 移除 health≤0 且死亡动画完毕的实体
              │     └─ 移除飞出屏幕的子弹
              │
              ├─ 2.6 胜负判定
              │     ├─ 所有僵尸死亡 + 所有波次结束 → Victory
              │     └─ 僵尸进入房子 → Defeat
              │
              └─ 2.7 输入处理（游戏特定）
                    ├─ 卡槽点击 → 进入种植模式
                    ├─ 种植模式 + 点击有效格子 → 种植植物
                    └─ 点击阳光 → 收集
```

## 7. 关键设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| **单例模式** | Game 为单例，管理器由其持有 | 避免全局变量，统一生命周期管理，各模块通过 `Game::GetInstance().GetXxxManager()` 访问 |
| **场景切换延迟** | pendingTransition 机制 | 避免在 Update 栈中直接销毁/重建场景导致的悬空指针 |
| **实体容器** | `std::vector<std::unique_ptr<T>>` | 自动内存管理，避免裸指针泄漏；插入/遍历性能满足需求（实体总数 < 200） |
| **碰撞检测** | 集中式 CollisionManager | 避免在各实体类中交叉引用，方便 AABB 和圆形碰撞的统一优化 |
| **坐标映射** | 地图 Map 统一提供 | 单一数据源，避免各实体自行计算导致不一致 |
| **资源路径** | 宏定义路径前缀 | 编译期拼接，零运行时开销；更换资源目录只需改宏定义 |
| **Delta Time** | 基于 `GetTickCount()` 或 `chrono` | 帧率无关的逻辑更新，FPS 波动不影响游戏速度 |
| **双缓冲** | EasyX `BeginBatchDraw/EndBatchDraw` | 消除画面闪烁，EasyX 原生支持 |
| **音频方案** | `mciSendString` | Windows 原生 API，无额外依赖，满足 WAV 播放需求 |
| **VS 项目结构** | 单个 vcxproj，include/src 分层 | 项目规模适中，无需拆分子项目；分层清晰利于维护 |
