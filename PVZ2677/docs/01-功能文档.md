# 植物大战僵尸 Lite — 功能文档（Functional Specification）

> 版本：v1.0-draft  
> 对应需求文档：EasyX 暑期课程大作业需求文档

---

## 1. 文档目的

本文档基于《EasyX 暑期课程大作业需求文档》，将用户需求拆解为可开发、可测试的功能点，明确每个模块的职责边界、输入输出、状态流转及异常处理策略，作为后续架构设计与代码实现的唯一功能依据。

---

## 2. 术语表

| 术语 | 定义 |
|------|------|
| **格 (Cell)** | 地图上的最小种植/战斗单元，固定为 5 行 × 9 列 |
| **阳光 (Sun)** | 游戏内核心资源，用于购买植物 |
| **卡槽 (Card Slot)** | UI 顶部显示可种植植物的区域，含冷却状态 |
| **波次 (Wave)** | 一组按时间或条件触发的僵尸生成事件 |
| **FPS** | 每秒帧数，目标 60 FPS |
| **Delta Time (dt)** | 相邻两帧之间的时间间隔，用于保证动画与逻辑的帧率无关性 |

## 4. 场景系统（Scene System）

### 4.1 启动界面（Splash Screen）

| 属性 | 说明 |
|------|------|
| **触发条件** | 程序启动后自动进入 |
| **持续时间** | 固定 2 秒，或用户点击/按键立即跳过 |
| **显示内容** | 项目名称、课程信息、开发团队 |
| **退场行为** | 淡入淡出过渡，结束后自动跳转主菜单 |
| **异常处理** | 资源加载失败则跳过，直接进入主菜单并记录日志 |

### 4.2 主菜单（Main Menu）

| 按钮 | 功能 |
|------|------|
| **开始游戏** | 跳转游戏界面，重置游戏状态，开始第一关 |
| **游戏说明** | 跳转帮助界面 |
| **设置** | 跳转设置界面（可为空页面占位） |
| **关于作者** | 跳转关于界面，显示开发者信息 |
| **退出游戏** | 安全释放资源，调用 `closegraph()`，正常退出进程 |

**交互细节：**
- 鼠标悬停按钮时：按钮背景色/边框高亮，播放悬停音效（可选）
- 鼠标点击时：播放点击音效，执行对应跳转
- 背景图固定，不得被拉伸变形（保持宽高比，黑边填充）

### 4.3 游戏界面（Gameplay）

**子区域划分：**

```
┌────────────────────────────────────────┐
│  [卡槽区] 向日葵 豌豆射手 坚果墙 ...     │  ← 顶部，高度固定

│                                        │
│           [战斗地图区] 5行×9列           │  ← 主体
│                                        │
├────────────────────────────────────────┤
│  [HUD区]  阳光: 150  金币: 0            │  ← 底部/左上角叠加
```

### 4.4 暂停界面（Pause Menu）

| 属性 | 说明 |
|------|------|
| **触发条件** | 游戏界面中按 `ESC` 键 |
| **表现** | 游戏画面变暗/半透明遮罩，暂停所有逻辑更新 |

| 属性 | 说明 |

## 5. 核心战斗系统

### 5.1 地图系统（Map）

#### 5.1.1 地图规格

| 属性 | 值 |
|------|-----|
| 行数 (ROW) | 5 |
| 列数 (COL) | 9 |
| 格子尺寸 | 根据窗口大小动态计算，保持正方形 |
| 房子位置 | 第 0 列左侧（x = 0 或固定偏移量） |
| 僵尸出生区 | 第 8 列右侧（或更右侧屏幕外） |

#### 5.1.2 地图元素

| 元素 | 描述 |
|------|------|
| **草坪** | 中间 5×9 区域，植物只能种植在此 |
| **房子** | 左侧边界，僵尸进入则游戏失败 |
| **道路** | 可选装饰，区分行 |
| **背景** | 底层贴图 |

#### 5.1.3 坐标映射

```cpp
// 屏幕坐标 ↔ 格子坐标 转换函数
CellPos ScreenToCell(int screenX, int screenY);
Point   CellToScreen(int row, int col);      // 返回格子中心点或左上角
bool    IsValidCell(int row, int col);       // 行列合法性校验
```

### 5.2 阳光系统（Sun System）

#### 5.2.1 阳光来源

| 来源 | 生成间隔 | 数量 | 行为 |
|------|---------|------|------|
| **自然阳光** | 每 10 秒（可调） | 25 | 从天空随机列飘落，停留 8 秒后消失 |
| **向日葵** | 每 24 秒 | 25 | 在向日葵所在格子上方生成，停留 8 秒 |

#### 5.2.2 阳光状态机

```
┌─────────┐  生成   ┌─────────┐  点击   ┌─────────┐
│  Spawning │ ─────►│ Floating │ ─────►│ Collected│
└─────────┘       └─────────┘       └─────────┘
                      │ 超时 8s
                      ▼
                   ┌─────────┐
                   │ Vanished│
                   └─────────┘
```

| 状态 | 说明 |
|------|------|
| **Spawning** | 生成瞬间，可能伴随缩放动画 |
| **Floating** | 可点击状态，向日葵产出的阳光停留原地，自然阳光匀速飘落 |
| **Collected** | 被点击后，飞向左上角 HUD，到达后阳光值 +25 |
| **Vanished** | 超时未被点击，自动消失 |

#### 5.2.3 阳光 HUD

- 位置：左上角固定坐标
- 显示：阳光图标 + 当前数值
- 更新时机：阳光被收集/消耗时立即刷新

#### 5.3.1 植物基类属性

| 属性 | 类型 | 说明 |
|------|------|------|

| 生命值 | 300 |
| 冷却 | 7.5 秒 |
| 攻击速度 | 0（不攻击） |
| 特殊能力 | 每 24 秒产出 25 阳光 |
| 动画 | 待机摇摆动画 |

#### 5.3.3 豌豆射手（PeaShooter）

| 属性 | 值 |
|------|-----|
| 价格 | 100 阳光 |
| 生命值 | 300 |
| 冷却 | 7.5 秒 |
| 攻击速度 | 1.5 秒/发 |
| 子弹伤害 | 20 |
| 动画 | 待机摇摆 + 攻击时头部伸长 |

#### 5.3.4 扩展植物（预留）

| 植物 | 特性概述 |
|------|---------|
| **坚果墙** | 高生命值（4000），阻挡僵尸前进，不攻击 |
| **寒冰射手** | 攻击附加减速效果（僵尸移速减半，持续 3 秒） |
| **双发射手** | 每次发射 2 颗豌豆 |

#### 5.3.5 植物状态机

```
┌──────────┐  种植完成   ┌──────────┐  有敌人同行   ┌──────────┐
│ Planting │ ─────────►│  Idle   │ ────────────►│ Attacking│
└──────────┘           └────┬─────┘              └────┬─────┘
                            │                         │
                            │ 同行无敌人               │ 攻击间隔到
                            ▼                         ▼
                      ┌──────────┐              ┌──────────┐
                      │   Idle   │◄─────────────│ Attacking│
                      └──────────┘              └──────────┘
                            │
                            │ health <= 0
                            ▼
                      ┌──────────┐
                      │   Dead   │ ──► 播放死亡动画后销毁
                      └──────────┘
```

#### 5.3.6 种植流程

1. 玩家点击卡槽中的植物卡片 → 判断阳光是否足够 & 卡片是否在冷却中
2. 若满足条件，鼠标变为植物预览图（半透明）
3. 玩家移动到有效格子，预览图变绿色；无效格子变红色
4. 点击有效格子 → 扣除阳光 → 卡片进入冷却 → 格子中生成植物（Planting 状态）
5. Planting 状态持续 0.5 秒后转为 Idle

| 波次 | 触发时间 | 僵尸配置 |

| 第 2 波 | 第 1 波结束后 15s | 普通僵尸 × 2 |
| 第 3 波 | 第 2 波结束后 15s | 普通僵尸 × 2 + 铁桶 × 1 |
| ... | ... | 逐步增加数量和种类 |
| 最终波 | 固定总波数（如 10 波） | 大量混合僵尸 |

### 5.5 子弹系统（Bullet System）

#### 5.5.1 豌豆子弹属性

| 属性 | 值 |
|------|-----|
| 速度 | 150 像素/秒（向右→左？不，植物在左，僵尸在右，子弹向右飞） |
| 伤害 | 20 |
| 大小 | 碰撞检测半径 10 像素 |
| 生命周期 | 飞出屏幕或命中后销毁 |

#### 5.5.2 子弹生命周期

```
生成（植物位置） → 匀速向右飞行 → 碰撞检测 → 命中僵尸（伤害结算+销毁）
                                              → 飞出屏幕边界（销毁）
```

### 5.6 战斗判定系统（Combat System）

#### 5.6.1 碰撞检测规则

| 碰撞对 | 判定条件 | 结果 |
|--------|---------|------|
| 子弹 vs 僵尸 | 圆形/矩形包围盒相交 | 子弹销毁，僵尸扣血 |
| 僵尸 vs 植物 | 僵尸 x 坐标 ≤ 植物所在格右边界 | 僵尸进入 Attacking 状态 |
| 僵尸 vs 房子 | 僵尸 x 坐标 ≤ 房子边界线 | 触发 GameOver |

#### 5.6.2 伤害结算

- 植物受击：health -= 僵尸.attackPower × dt（按时间累加，或按攻击间隔结算）
- 僵尸受击：health -= 子弹.damage（即时）
- 死亡判定：health ≤ 0 时触发死亡动画，动画结束后对象销毁

## 6. 动画系统（Animation System）

### 6.1 帧动画通用接口

```cpp
class Animation {
    // 属性
    vector<IMAGE*> frames;      // 帧序列
    float frameDuration;         // 每帧持续时间（秒）
    float currentTime;           // 当前帧已播放时间
    int currentFrame;            // 当前帧索引
    bool isLooping;              // 是否循环
    bool isFinished;             // 是否播放完毕（非循环）

    // 方法
    void Update(float dt);
    void Render(int x, int y);
    void Play();
    void Stop();
    void Reset();
};
```

### 6.2 各对象动画清单

| 对象 | 动画状态 | 帧数 | 循环 |
|------|---------|------|------|
| 向日葵 | 待机摇摆 | ≥2 | 是 |
| 豌豆射手 | 待机摇摆 | ≥2 | 是 |
| 豌豆射手 | 攻击伸长 | ≥2 | 否（攻击时触发一次） |
| 普通僵尸 | 行走 | ≥2 | 是 |
| 普通僵尸 | 攻击 | ≥2 | 是 |
| 普通僵尸 | 死亡 | ≥3 | 否 |
| 铁桶僵尸 | 同普通 + 铁桶掉落 | - | - |
| 阳光 | 旋转/闪烁 | ≥2 | 是 |
| 子弹 | 飞行 | 1（静态图） | - |
| 按钮 | 悬停高亮 | 1（变色/缩放） | - |
| 爆炸 | 粒子扩散 | ≥3 | 否 |

|---------|---------|------|
| UI 图片 | 启动时 | 预加载，常驻内存 |
| 植物/僵尸动画帧 | 首次进入游戏界面 | 按需加载，缓存 |
| 背景音乐 | 进入对应场景时 | 流式播放 |
| 音效 | 首次触发时 | 预加载常用音效，懒加载非常用 |

### 8.3 资源路径配置

- 所有路径通过 `ResourceManager` 统一管理
- 禁止在代码中硬写绝对路径
- 支持相对路径：`Resource/images/plants/...`

## 11. 异常与边界处理

| 场景 | 预期行为 |
|------|---------|
| 图片资源缺失 | 使用纯色矩形占位，输出错误日志，不崩溃 |
| 音频资源缺失 | 静默跳过，不崩溃 |
| 配置文件缺失 | 使用默认配置，自动创建新配置文件 |
| 阳光不足时点击卡片 | 播放错误提示音（或视觉反馈），不执行种植 |
| 冷却中点击卡片 | 同上也 |
| 非草坪格子点击种植 | 预览图变红，点击无效 |
| 已有植物的格子点击 | 预览图变红，点击无效 |
| 窗口最小化/失去焦点 | 自动暂停游戏（可选） |
| 帧率骤降（<30 FPS） | 游戏逻辑仍基于 dt 正确运行，可能跳帧 |
